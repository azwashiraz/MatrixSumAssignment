# MatrixSumAssignment

ASSIGNMENT 1

DESCRIPTION 

This program works with a matrix. It stores the matrix values and performs different calculations on them. The program calculates the sum of all rows and the sum of all columns separately. It also finds the sum of the main diagonal (forward diagonal) and the reverse diagonal of the matrix.

This program shows how two dimensional arrays and loops can be used in Java to process and calculate values in a matrix.

ASSIGNMENT 2

DISCRIPTION

This program works with a square matrix. First, it asks the user to enter the dimension of the matrix (for example 3×3 or 4×4). After that the user enters all the elements of the matrix.

Once the matrix is filled the program calculates the sum of elements in each row and the sum of elements in each column. It also calculates the sum of the main diagonal (from top-left to bottom-right) and the sum of the reverse diagonal (from top-right to bottom-left).

This program demonstrates how square matrices can be handled in Java using two dimensional arrays, loops, and basic calculations.


ASSIGNMENT 3

DESCRIPTION:

This project contains Java programs that convert a decimal number into its binary form. The work is divided into three parts.

In Part (a): the program takes an integer input from the user and converts it into binary using a simple basic method.

In Part (b): a user-defined function is used which takes an integer as a parameter, converts it into binary, stores the binary digits in an integer array, and returns that array.

In Part (c), another user-defined function performs the same task but stores the binary digits in a string and returns the binary number as a string. All programs are written in a simple and beginner-friendly way without using complex data structures.


ASSIGNMENT 4

DESCRIPTION:

This project contains Java programs that read sales data from a file (SalesEVESession.txt) and calculate totals based on different categories using arrays and HashMap.

Product Wise Sum (No HashMap):
This program calculates total sales for a fixed list of products using arrays. It reads the file, matches product names with a predefined list, and updates their totals. It is a basic approach and does not use HashMap.

ID Wise Sum (No HashMap):
This program calculates total sales based on fixed representative IDs using arrays. It matches IDs from the file with predefined values and adds their sales totals. It is simple and does not use HashMap.

Product Wise Sum (Using HashMap):
This program uses a HashMap to calculate product-wise sales totals dynamically. It reads product names from the file and automatically adds new entries if needed. This method is more flexible and efficient.

ID Wise Sum (Using HashMap):
This program calculates sales totals based on representative IDs using a HashMap. It dynamically stores IDs and updates their values. This approach is more efficient and works without fixed limits.
